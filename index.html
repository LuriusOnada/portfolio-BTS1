Le code :
Dans ce projet que Kevin TARTIERE et moi avons réalisé, dont le but premier était :
Réalisez un site ou une application permettant de localiser l'utilisateur afin que ce dernier puisse par la suite placer des points de repère, marqueur, ici pour noter la localisation GPS de l'endroit où il avait garé sa voiture.

L'application :
Afin de le réaliser, nous avons utilisé l'API (Applications Programming Interface) de Google Maps, ainsi que les langages HTML (HyperText Markup Language), CSS (Cascading Style Sheet) et JS (JavaScript), avec la bibliothèque JQuery.
Pour le confort et la continuité visuelle, la carte qui s'affiche avec les paramètres de la version nocturne de Google Maps.

Sur la page principale trois boutons sont présents :
Le premier, nommer "Save this location" permet de faire apparaitre un popup demandant le nom du marqueur qui s'affichera à l'écran.
Toutefois, il faut faire attention, plusieurs marqueurs peuvent se retrouver à la même position et donc leurs noms peuvent se mélanger.

Le second, qui contient une icône de boussole sert simplement de recentrer la carte sur la position actuelle de l'utilisateur
Le dernier, nommer "Show markers" ouvre une seconde page, dans laquelle l'on retrouve la liste des marqueurs créés, mais où on peut également les gérer.

Ainsi, sur cette seconde page, il y a certes deux boutons pour gérer les marqueurs, le premier, "Renommer", permet de renommer un marqueur déjà créé et cela grâce à un popup similaire à celui permettant d'en créer un.

Et à sa gauche, un bouton "Supprimer", qui comme son nom l'indique supprime les informations du marqueur en question.
De plus en bas de page, un nouveau bouton et présent, "Return to map", et nous permet de quitter cette page de gestion des marqueur afin de revenir sur la carte.

Le code :
Beaucoup d'élément dans le code s'explique facilement avec la partie précédente ou parce qu'il découle de l'API de Google Maps.
Toutefois, quelques lignes mérite des explications, notamment sur le stockage d'information, les cookies.

Ici la fonction setCookie, s'actualise à chaque fois que l'utilisateur ouvre son navigateur, cela permet de conserver pendant une semaine les données stocker et ce depuis la dernière ouverture de la page.
La fonction JSON.stringify, permet quand à elle de transformer en string les informations contenu dans "cookie", soit le nom et les coordonnées associés de chaque marqueur, en plus d'un id pour l'identifié dans le tableau ainsi créée, afin de le modifié ou le supprimer.
La fonction getCookie est ainsi ici pour ajouter à chaque nouveau marqueur une nouvelle "ligne" au tableau cookie.
